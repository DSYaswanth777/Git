Git Commands

git init

To commit new changes in side any file or folder

git commit -a -m "any required message "

and then git push origin master command to reflect changes

While updating already existing file in git

git add .

git commit -m "required message"

git push origin master

To remove any cached files which present in bin folder which may generated while program runtime by using below command we can remove only in git bash or git repo but not in local machine

git -rm cached '.classpath'

So in case we wanted to remove from the staging area all files from specific directory, we have to

git rm -r --cached bin/

it removes all files from our specfic directory

To ignore such files every time we don;t need to type such command every time

To overcome this we use .gitignore in that we specificy what we want ignore every time to specific a file use .extension file name u need to specific directory use name/

if we don't want make any changes in commit message then use git commit --amend --no--edit to commit changes

git restore --staged .. to unstage which is present in already present in staged area

git is a version control

To create a new branch

git branch creates new branch

To know the branches which are in remote and which are in local

git branch -r (to know branches which are in remote)

git branch -a to know all branches remote and local

To change branch

git check out -b

to get back our master branch we use git checkout master

TO remove specific branch

git branch -d branch name

switch branches is a new update in git

git switch master (we can switch to new branch

git switch -c new_branch (with this we can create new branch switch to that

git switch - (used to previous branch)
